## 0. 부재(Member) 생성 기본 원칙

Step 적합 안정 조건을 검사하기 전에, 입력 데이터의 부재가 올바르게 정의되어 있는지 검증해야 한다.

### 0.1 부재 정의 규칙

| 규칙 | 설명 |
|------|------|
| **그리드 정렬** | 부재의 길이가 x, y, z 축과 평행한 인접 절점으로 정의되어야 함 |
| **방향 벡터** | 임의의 2개 부재의 설치 방향 vector의 cosine similarity는 **0 또는 1** (x, y, z 축 중 하나와 평행) |
| **수평 부재** | node_i와 node_j는 x 또는 y 값 **둘 중 하나만** 변화 (대각선 부재 불허) |
| **수직 부재** | node_i와 node_j의 (x, y) 좌표는 동일, z 좌표만 상이 |
| **바닥층 레벨** | z 좌표의 최소값은 모두 동일해야 함 (1층 기둥은 모두 같은 바닥층 레벨) |

### 0.2 예외(오류) 케이스 검출

| 오류 유형 | 설명 |
|----------|------|
| **Orphan 노드** | 어떤 element와도 연결되지 않은 절점 |
| **중복 ID** | 동일한 ID를 가진 node 또는 element |
| **길이 0 부재** | node_i와 node_j가 완전히 동일한 좌표 |
| **중복 부재** | 완전히 동일한 공간에 겹쳐서 배치된 부재 |

### 0.3 검증 순서

```
입력 데이터 검증
    │
    ├── 1. 노드 검증
    │   ├── 중복 ID 검사
    │   └── 바닥층 레벨 일관성 검사
    │
    ├── 2. 부재 검증
    │   ├── 중복 ID 검사
    │   ├── 길이 0 부재 검사
    │   ├── 중복 부재 검사
    │   └── 그리드 정렬 검사 (축과 평행)
    │
    └── 3. 연결 검증
        └── Orphan 노드 검사
```

## 1. 핵심 개념: 개별 부재 vs 패턴

### 1.1 두 가지 다른 검증 단계

| 단계 | 대상 | 목적 | 예시 |
|------|------|------|------|
| **설치 가능 후보** (Candidate Eligibility) | 개별 부재 | "이 부재를 설치할 수 있는가?" | 기둥이 지상층이거나 하부 기둥 위에 적층 |
| **Step 적합 안정 조건** (Pattern Stability) | **패턴 (부재 조합)** | "이 패턴이 구조적으로 자립 가능한가?" | 기둥→기둥→거더→거더 패턴이 인접 구조물에 연결됨 |

### 1.2 핵심 원칙

> **Step 적합 안정 판정은 개별 부재가 아니라 패턴 단위로 수행한다.**

- 기둥 1개가 설치 가능하다고 해서 Step이 되는 것이 아님
- 패턴이 완성되어야만 Step으로 인정됨
- 패턴은 인접 구조물과의 연결 관계에 따라 적합 여부가 결정됨

## 2. 용어 정의

| 용어 | 정의 |
|------|------|
| **Sequence** | 단위 시간. N개 workfront가 각각 1개씩 부재를 동시에 설치 |
| **Step** | **패턴 기반 적합 안정 조건**을 만족하는 부재 그룹 |
| **패턴** | 부재 설치 순서의 조합 (예: 기둥→거더, 기둥→기둥→거더→거더) |
| **인접 구조물** | 이미 설치 완료된 기존 구조 (Bootstrap 또는 이전 Step들) |
| **국소 안정 문맥** | 후보 패턴의 node_i/node_j 에 실제로 닿는 이미 안정 판정 통과 부재들의 부분집합 |

**핵심**: Step 수 << Sequence 수  
예) 270 Sequences → 약 50-80 Steps가 정상

**Sequence 해석 보강**:
- Sequence 번호는 설치 순번이 아니라 **동시 설치 라운드** 다.
- 같은 Sequence 에서 여러 workfront 가 동시에 설치되면 동일한 sequence 번호를 공유한다.
- 따라서 같은 Sequence 안에서 workfront A 는 1층, workfront B 는 2층에서 동시에 작업할 수 있다.

## 3. Step 적합 안정 조건 — 패턴별 판정 규칙

### 3.1 Bootstrap (첫 번째 Step) — 독립 존재 가능

**패턴**: `기둥→기둥→거더→기둥→거더` (5개 부재)

**조건**:
- 지상층(z=0) 기둥 3개
- 기둥 상단(node_j) 연결 거더 2개
- 두 거더가 **90도 직교** (dot product = 0)

```
    (평면도 - 위에서 본 모습)
    
    C2
    │
    G1
    │
    C1 ───G2─── C3
```

- **C1**: 중앙 기둥 (node_j = 거더 교차점)
- **G1**: C1 상단에서 Y방향으로 C2 연결
- **G2**: C1 상단에서 X방향으로 C3 연결
- **90도 직교**: G1 방향벡터 · G2 방향벡터 = 0

**검증**: `has_minimum_assembly()` — 독립적인 구조물로 존재 가능

---

### 3.2 부재 1개 패턴

| 패턴 | 적합 안정 조건 | 발생 상황 |
|------|---------------|----------|
| `(거더)` | **이미 설치된 두 기둥**을 연결할 때만 PASS | 어떤 층의 마지막 **폐합 부재**로 발생 |

**핵심**: 단독 거더는 양쪽 노드가 **기존에 설치된 기둥**에 연결되어야만 함

---

### 3.3 부재 2개 패턴

| 패턴 | 적합 안정 조건 | 비고 |
|------|---------------|------|
| `(기둥→기둥)` | `(기둥→기둥→거더→거더)`의 **sub pattern** | 단독으로는 불완전, 후속 거더 필요 |
| `(기둥→거더)` | **인접 구조물의 기둥 node_j와 거더로 연결**될 때 PASS | 빈번하게 발생하는 확장 패턴 |
| `(거더→거더)` | `(기둥→기둥→거더→거더)`의 **sub pattern** | 단독으로는 불완전 |

---

### 3.4 부재 3개 패턴

| 패턴 | 적합 안정 조건 | 비고 |
|------|---------------|------|
| `(기둥→기둥→기둥)` | **FAIL — 어떤 경우에도 불허** | 3개 연속 기둥은 횡방향 지지 없음 |
| `(기둥→기둥→거더)` | `(기둥→기둥→거더→거더)` 또는 `(기둥→기둥→거더→기둥→거더)`의 **sub pattern** | 단독으로는 불완전 |
| `(기둥→거더→기둥)` | `(기둥→거더→기둥→거더)`의 **sub pattern** | 단독으로는 불완전 |
| `(기둥→거더→거더)` | **캔틸레버 상태 검사 필수** | 두 거더가 모두 지지되어야 PASS |

**`(기둥→거더→거더)` 상세 조건**:
- 첫 번째 거더: 새 기둥과 인접 구조물 기둥 연결
- 두 번째 거더: **캔틸레버가 아니어야 함** (양쪽 노드 모두 기둥에 지지)

## 4. 층별 우선 완료 규칙 (Floor Completion Priority)

### 4.1 규칙

> **각 층의 미설치 부재가 5개 이하인 경우, 해당 층의 부재를 우선적으로 설치한다.**

### 4.2 문제 상황 (규칙 미적용 시)

- 1층이 완료되지 않았는데 2층 작업율이 급격히 상승
- 하층부 미완료 상태에서 상층부 작업 집중 → 비현실적 시공 순서

### 4.3 적용 조건

| 조건 | 설명 |
|------|------|
| **임계값** | 층별 미설치 부재 ≤ 5개 |
| **우선순위** | 해당 층 부재가 후보 선정에서 가중치 상향 |
| **검사 시점** | 매 Sequence 시작 전 층별 잔여 부재 수 계산 |

### 4.4 플로우

```
Sequence 시작
    │
    ▼
층별 미설치 부재 수 계산
    │
    ▼
미설치 ≤ 5개인 층 존재?
    │
    ├── YES → 해당 층 부재 우선 설치
    │
    └── NO → 일반 후보 선정 진행
```

## 5. 적합 안정 조건 판정 플로우

```
workfront 확장 후보 생성
    │
    ▼
패턴 생성 (2~5개 부재 조합)
    │
    ▼
후보 패턴 endpoint 와 닿는 국소 안정 문맥 추출
    │
    ├── 국소 안정 문맥 있음
    │     └── 확장 패턴으로 평가
    │
    └── 국소 안정 문맥 없음
          └── 독립 bootstrap 가능성 평가
    │
    ▼
┌─────────────────────────────────┐
│   Step 적합 안정 조건 검사       │
│                                 │
│  1. 금지 패턴 체크               │
│     - (기둥→기둥→기둥) → FAIL   │
│     - (기둥→기둥→기둥→거더) → FAIL │
│                                 │
│  2. 패턴별 조건 검사             │
│     - Sub pattern인가?          │
│       → 상위 패턴 완성 대기      │
│     - 완성 패턴인가?             │
│       → 국소 안정 문맥 기준 연결 검사 │
│                                 │
│  3. 인접 구조물 연결 검사        │
│     - 거더가 국소 stable column/girder 에 연결? │
│     - 새 기둥이 stable 에 anchor 되었는가? │
│     - 캔틸레버 상태 아닌가?      │
│                                 │
└─────────────────────────────────┘
    │
    ▼
PASS → Step 생성 (가중치 적용 종합점수 평가)
FAIL → 패턴 확장 또는 롤백
```

## 6. 패턴 연속 적용 예시

인접 구조물에 연결하여 확장할 때:

```
[(기둥→거더→기둥→거더) + (기둥→거더)]
```

**해석**:
1. 첫 번째 패턴 `(기둥→거더→기둥→거더)`: 4개 부재로 안정 구조 형성
2. 두 번째 패턴 `(기둥→거더)`: 인접 구조물(첫 번째 패턴의 기둥)에 연결

## 7. 금지 패턴 정리

| 패턴 | 사유 |
|------|------|
| `기둥→기둥→기둥` | 3개 연속 기둥은 횡방향 지지 없음 — **절대 불허** |
| `기둥→기둥→기둥→거더` | 3기둥 후 거더 1개로는 불안정 — **절대 불허** |

## 8. Sub Pattern vs 완성 패턴

### 7.1 Sub Pattern (단독 불가, 상위 패턴 완성 필요)

| Sub Pattern | 상위 패턴 |
|-------------|----------|
| `(기둥→기둥)` | `(기둥→기둥→거더→거더)` |
| `(거더→거더)` | `(기둥→기둥→거더→거더)` |
| `(기둥→기둥→거더)` | `(기둥→기둥→거더→거더)` 또는 `(기둥→기둥→거더→기둥→거더)` |
| `(기둥→거더→기둥)` | `(기둥→거더→기둥→거더)` |

### 7.2 완성 패턴 (인접 구조물 연결 시 Step 가능)

| 완성 패턴 | 조건 |
|----------|------|
| `(거더)` | 이미 설치된 두 기둥 연결 (폐합) |
| `(기둥→거더)` | 인접 구조물 기둥 node_j에 거더 연결 |
| `(기둥→거더→거더)` | 캔틸레버 아님 확인 필수 |
| `(기둥→기둥→거더→거더)` | 인접 구조물 연결 |
| `(기둥→거더→기둥→거더)` | 인접 구조물 연결 |
| `(기둥→기둥→거더→기둥→거더)` | **독립 존재 가능** (Bootstrap) |

## 9. 현재 구현 상태

### 9.1 반영 완료 항목

1. **Sequence-driven parallel installation**
    - 각 workfront 는 같은 Sequence 라운드에서 최대 1개 부재를 선택한다.
    - 같은 라운드에 설치된 여러 부재는 동일한 sequence 번호를 공유한다.

2. **Pattern-buffered Step generation**
    - workfront 별 버퍼에 Sequence 결과를 누적하고, 완성 패턴일 때만 Step 을 방출한다.
    - Sequence 1회마다 Step 1개를 기계적으로 만들지 않는다.

3. **국소 안정 문맥 기반 평가**
    - Step 적합 안정 조건 검사는 stable 전체를 매번 재검사하지 않는다.
    - 후보 패턴과 실제로 닿는 stable 부재만 추출하여 국소 문맥으로 사용한다.

4. **독립 bootstrap 예외 유지**
    - 국소 안정 문맥이 비어 있어도, 후보 패턴 자체가 독립 bootstrap 최소 조립체이면 PASS 가능하다.

5. **상층 확장 연결 보정**
    - 새 거더가 직접 stable endpoint 에 닿지 않더라도, stable 에 anchor 된 새 기둥 상단을 통해 연결되는 경우는 extension 으로 허용한다.

6. **Global Step Cycle 집계 + LocalStep 병합 반영**
    - Step 생성은 workfront별 즉시 방출이 아니라 cycle 집계 방식으로 변경되었다.
    - 각 cycle 내부 sequence round 에서 workfront 당 최대 1개 부재만 선택한다.
    - workfront 버퍼가 완성 패턴 + 안정 조건 pass 를 만족하면 `LocalStep` 으로 cycle 수집 버퍼에 기록한다.
    - 같은 cycle 내에서 local step 생성 성공 workfront 는 남은 라운드에서 제외한다.
    - cycle 종료 시 수집된 `LocalStep` 들을 1개의 `SimStep` 으로 병합한다 (`local_steps` 보존).

7. **Sequence 번호 부여 규칙 고정 (Round-Robin Collation)**
    - sequence 번호는 1부터 시작하는 global 연속 번호를 사용한다.
    - 하나의 global step 내부에서는 round-robin collation 으로 sequence 를 구성한다.
    - 같은 round 에 설치된 부재들은 동일 sequence 번호를 공유한다.
    - 결과적으로 `Sequence != Step` 가정이 유지되며, multi-workfront 에서 step 수는 sequence 수보다 작아야 정상이다.

## 10. 구현 원칙 요약

1. **전역 재검사 금지**: 매 Sequence 마다 stable 전체를 다시 검사하지 않는다.
2. **신규 부재만 검사도 금지**: 신규 패턴만 떼어 검사하면 인접 안정구조를 고려할 수 없다.
3. **정답은 국소 문맥 평가**: 신규 패턴 + 실제 인접 stable 부재만 합쳐서 평가한다.
4. **Disconnected case 예외**: 국소 문맥이 비어 있으면 독립 bootstrap 규칙으로 판정한다.
5. **Step 방출 조건**: 완성 패턴 + 적합 안정 조건 PASS 일 때만 LocalStep 생성하고, cycle 종료 시 병합 SimStep 을 방출한다.
6. **Sequence 규칙 유지**: sequence 는 global 1-based 연속 번호를 유지하고 round 동시 설치는 동일 번호를 공유한다.
