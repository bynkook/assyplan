# 2026-03-20 Patch — Sequence 뷰 Cycle 기반 정렬 전환

## 문제

Simulation Sequence 뷰에서 지지 기둥이 거더보다 늦게 표시되어 캔틸레버처럼 보이는 현상 발생.

- 시나리오 1, seq 30: 거더 952 (WF2, cycle 17 승인) — 지지 기둥 272 (WF8, cycle 16 승인)가 seq 36에서야 표시
- 시나리오 1, seq 31: 거더 642 (WF1, cycle 20 승인) — 지지 기둥 103 (WF9, cycle 19 승인)가 seq 34에서야 표시

## 근본 원인

`build_sim_sequence_rows` (lib.rs)가 WF 이력을 position 기반으로 interleave하고 있었음. WF마다 local step 크기가 다르므로(Bootstrap 5개 vs 단독 Girder 1개) 같은 position index가 서로 다른 cycle에 매핑됨. 결과적으로 먼저 승인된 기둥이 나중에 표시되는 역전 발생.

엔진의 안정성 판정 로직(`check_step_bundle_stability`, `validate_girder_support`)은 정상 — cycle_stable_context 기준으로 양단 기둥 지지를 정확히 검증하고 있었음.

## 수정 내용

### 코드 변경: `src/rust/src/lib.rs` — `build_sim_sequence_rows`

기존(position 기반):
- 각 WF 이력을 flat vector로 펼친 뒤 max_len까지 position index로 순회
- 같은 position의 각 WF 원소를 하나씩 꺼내 seq 부여

변경(cycle 기반):
- `scenario.steps`를 순서대로 순회
- 각 step의 `local_steps`에서 WF별 원소를 `BTreeMap<wf_id, Vec<(floor, eid)>>`로 수집
- cycle 블록 내에서 position interleave하여 seq 부여
- cycle N 원소가 반드시 cycle N+1 원소보다 앞에 표시됨을 보장
