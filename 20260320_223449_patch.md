# Patch: Simulation Sequence View를 WF 연속 이력 기준으로 정정

날짜: 2026-03-20 22:34:49
대상 프로젝트: AssyPlan
빌드 검증: cargo build --release 성공, 루트 assyplan.exe 갱신 완료

## 변경 배경

사용자 정본 정의를 기준으로 Simulation Sequence 뷰 의미를 확정했다.

- 각 WF의 sequence는 해당 WF가 승인받은 local step들의 element_ids를 생성 순서대로 이어붙인 연속 리스트다.
- Sequence 뷰는 모든 WF sequence에서 같은 인덱스의 원소를 하나씩 꺼내 누적으로 보여준다.
- Step 뷰는 안정성 패턴 단위 뷰다.
- 따라서 Step과 Sequence는 다른 의미이며 서로의 표현 모델을 섞으면 안 된다.

기존 구현은 SimStep 내부에 round-robin으로 재조합한 sequences를 만들고, Simulation Sequence 뷰와 export가 그것을 source of truth로 사용했다. 이 때문에 WF 로컬 순서를 보존해야 하는 Sequence 뷰에서 local step 내부가 다른 WF와 섞여 보이는 문제가 있었다.

## 핵심 수정 사항

### 1. round-robin 기반 SimStep sequence 생성 제거

파일:
- src/rust/src/graphics/ui.rs

변경 내용:
- SimStep에서 sequences 필드를 제거했다.
- from_elements, from_sequences 헬퍼를 제거했다.
- SimStep::from_local_steps(local_steps, start_seq)를 SimStep::from_local_steps(local_steps)로 단순화했다.
- global step의 element_ids는 local_steps를 순서대로 평탄화한 결과로 유지했다.

의도:
- Step 데이터 구조에서 round-robin sequence 재조합을 제거해, Sequence 뷰가 잘못된 중간 표현을 참조하지 못하게 했다.

### 2. Simulation Sequence 뷰를 WF history 기반으로 재구성

파일:
- src/rust/src/lib.rs

추가:
- SimSequenceRow 내부 헬퍼 구조체 추가
- build_sim_sequence_rows(scenario) 추가
- installed_ids_through_sim_sequence(scenario, current_sequence) 추가

동작:
- scenario.steps를 순서대로 순회한다.
- step.local_steps의 workfront_id별로 element_ids를 이어붙여 WF 연속 이력을 만든다.
- 전역 sequence N은 각 WF history의 N번째 원소를 하나씩 꺼낸 집합으로 재구성한다.
- Sequence 뷰는 sequence_number <= current_sequence 기준이 아니라, 위 규칙으로 구성된 WF history 기반 누적 결과를 사용한다.

효과:
- WF 내부 설치 순서가 그대로 유지된다.
- Sequence 뷰가 Step 내부 조합 규칙을 잘못 재사용하지 않는다.

### 3. Simulation sequence 정보 표시 수정

파일:
- src/rust/src/lib.rs

변경 내용:
- 상단 detail bar의 Seq 정보가 WF history 기반 current sequence 항목을 사용하도록 수정했다.
- 현재 sequence에서 실제로 새로 설치된 WF별 element만 표시한다.

### 4. Simulation debug export의 sequence table 수정

파일:
- src/rust/src/lib.rs

변경 내용:
- sim_sequence_table_scene_XXXX.csv 생성 시 step.sequences를 사용하던 경로를 제거했다.
- export도 build_sim_sequence_rows(scenario) 기반으로 sequence_order, workfront_id, element_id, member_type, step을 작성한다.

의도:
- UI와 export의 sequence 의미를 동일하게 맞췄다.

### 5. sim_engine 단순화 및 로그 정합성 보정

파일:
- src/rust/src/sim_engine.rs

변경 내용:
- next_sequence_start 제거
- SimStep::from_local_steps 호출부를 새 시그니처로 변경
- cycle end trace의 step_rounds는 local_steps 개수 같은 잘못된 값이 아니라 실제 cycle_round_index를 기록하도록 유지

주의:
- SimSequence 타입 자체는 WorkfrontState.buffer_sequences에서 엔진 내부 버퍼 표현에 여전히 사용하므로 제거하지 않았다.
- 제거된 것은 SimStep level의 round-robin sequence 표현이다.

## 결론

이번 수정은 simulation approval 규칙을 임의로 바꾸지 않고, Sequence 뷰의 source of truth를 사용자 정의와 일치시키는 방향으로 정리했다.

즉:
- Step은 안정성 패턴 단위
- Sequence는 WF별 연속 작업 이력
- Sequence 뷰는 모든 WF에서 하나씩 꺼내 누적 표시

이 구분을 코드, export, 문서에 일관되게 반영했다.
