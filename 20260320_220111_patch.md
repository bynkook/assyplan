## 배경

7x21x4 그리드 + 10 Workfront 테스트에서 3가지 문제 발견:

1. **시나리오 1 실패** (MaxIterations) — WF5가 floor 3에서 rollback-retry 무한 루프
2. **시나리오 3 특정 영역 step 지연** — 기둥 32,52,53 위치에 step 미생성
3. **시나리오 3 sequence 1에서 캔틸레버 거더** — 484번 거더가 지지 기둥 없이 표시

## 근본 원인 분석

### Issue 3 근본 원인: `has_stable_structure` 오판정

`classify_member_signature()`의 패턴 분류 분기점인 `has_stable_structure`가 잘못된 범위로 계산되고 있었다.

**변경 전**: `has_stable_structure = !support_ids.is_empty()`
- `support_ids = stable_ids + cycle_local_steps(모든 WF) + buffer`
- 다른 WF의 미커밋(locally completed but not globally committed) 부재를 "이미 설치된 구조"로 취급
- Bootstrap cycle에서 `stable_ids`가 비어있어도 다른 WF의 `cycle_local_steps`가 있으면 `true`
- 결과: 빈 버퍼에서 거더 선택 가능 → `"G"` → `Complete(Girder)` → 단일 거더 LocalStep 생성

**devplandoc.md 위반**: "`(거더)`: **이미 설치된** 두 기둥을 연결하는 폐합 부재일 때만 pass" — `cycle_local_steps`의 미커밋 부재는 "이미 설치된(globally committed)" 구조가 아니다.

### Issue 1 근본 원인: 교착 탈출 조건 부재

`cycle_no_local_step_rounds` 변수가 추적되지만 이를 기반으로 cycle을 종료하는 break 조건이 없었다. Column 설치가 `cycle_no_progress_count`를 리셋하지만, LocalStep 생성 없이 설치↔rollback만 반복하는 상황을 감지하지 못함.

### Issue 2

Issue 1, 3의 결과적 현상. 근본 원인 수정 시 개선 예상.

### Phase B: 교착 탈출 조건 추가

```rust

    break; // Stagnation within cycle

// ↓ 추가

    break; // Deadlock: no local step produced for too long

```

Threshold: `wf_count × 10` rounds 연속으로 어떤 WF도 LocalStep을 완성하지 못하면 cycle 종료.

### Phase C: rollback 카운터 로그 추가

- `cycle_rollback_count: u32` 변수 추가 (cycle 시작 시 0으로 초기화)
- 3곳의 rollback 경로에서 카운팅:
  1. pre-round Invalid pattern rollback
  2. no-candidate locked floor rollback
  3. infeasible emission rollback
- `sim.cycle.end` 이벤트에 `rollback_count` 필드 추가

## 수정 후 기대 동작

### Bootstrap cycle (`stable_ids` = empty)

- 모든 WF: 빈 버퍼 → `classify_member_signature("", false)` → `Incomplete(ColumnOnly)` → 첫 후보 반드시 Column
- 단일 거더 `Complete(Girder)` 판정 불가 → 캔틸레버 방지
- 모든 WF가 `CCGCG` Bootstrap 패턴으로 완성

### 후속 cycle (`stable_ids` non-empty)

- 단일 거더 가능: 지지 기둥이 `stable_ids`(이전 step, 이미 commit됨)에 포함 → 3D View에서 표시됨 → 캔틸레버 없음
- `check_bundle_stability`는 여전히 `support_ids`(cycle_local_steps 포함)로 검증 → 다중 부재 패턴의 인접 확장 정상 동작

### 교착 방지

- WF5 등이 floor 3에서 rollback-retry 반복 시, `wf_count × 10` rounds 후 cycle 종료
- `cycle_local_steps`가 비어있으면 `consecutive_empty_cycles += 1` → 5회 연속 시 `NoCandidates` 종료
